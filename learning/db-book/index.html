全书结构

先把数据库看成“一个 SQL 请求如何变成页 I/O”，再向下钻索引和存储，向上回到优化器、事务、恢复与分布式系统。

I查询语言先知道你问了什么
ready01SQL：声明式接口SQL 为什么不是“怎么做”，而是“我要什么”。 ready02关系模型与代数优化器真正操作的那门小语言。
II存储与索引把逻辑请求落到字节
ready03磁盘、文件、Buffer Pool页、slot、eviction，把“表”拆回物理结构。 ready04代价模型与索引选择为什么数据库最关心的不是 CPU，而是 I/O。 ready05B+ 树：十亿行里一步命中插入、分裂、范围扫描、fan-out、B+Tree vs LSM。
III查询执行把计划跑成数据流
ready06排序与哈希外排序、Grace Hashing，以及为什么内存不够才有趣。 ready07Joins 与 Iterator 模型Nested loop 到 hash join，统一成 next() 的数据流。 ready08查询优化器枚举计划、估计选择率、在指数空间里找便宜路径。
IV事务系统让并发和故障都像没发生
ready09事务、锁与隔离Serializability、2PL、deadlock、MVCC 的共同问题。 ready10Recovery 与 ARIESWAL、redo/undo、checkpoint，崩了也能回来。
V分布式与现代系统从单机 DB 到数据平台
ready112PC、Paxos 与分布式事务一旦机器不止一台，commit 就变成协议。 ready12NoSQL、Spark 与现代系统把 CS186 的古典内核接到 LSM、lakehouse、OLAP。
